# KOMI ARTMAKE サイト公開手順

## 全体の流れ

```
ドメイン取得 → GitHub登録・アップ → Cloudflare Pages設定 → ドメイン接続 → 完成
```

所要時間：約1〜2時間（作業自体は30分、DNS反映待ち最大48時間）

---

## STEP 1｜ドメイン取得（お名前ドットコム）

1. [お名前.com](https://www.onamae.com/) にアクセス
2. 検索欄に `artmake-komi` と入力して検索
3. `artmake-komi.com` にチェックを入れてカートへ
4. 初年度無料〜160円キャンペーンを確認して購入
5. **ネームサーバー設定は後でやるのでスキップでOK**

> 💡 更新費は年約1,500円。自動更新をONにしておくと失効しない。

---

## STEP 2｜GitHubアカウント登録

1. [github.com](https://github.com) にアクセス
2. 「Sign up」からアカウント作成（メール・パスワード・ユーザー名）
3. メール認証を完了

---

## STEP 3｜GitHubリポジトリ作成

1. GitHubにログイン後、右上の「+」→「New repository」
2. 以下の設定で作成：

| 項目 | 設定 |
|------|------|
| Repository name | `artmake-komi` |
| Visibility | **Public** |
| Add README | Off |
| Add .gitignore | Off |
| Add license | Off |

3. 「Create repository」をクリック

---

## STEP 4｜ファイルをGitHubにアップ

### 方法A：ブラウザから（Git不要・おすすめ）

1. 作成したリポジトリのページを開く
2. 「uploading an existing file」をクリック
3. ファイルをドラッグ&ドロップ

アップするファイル・フォルダ構成：
```
artmake-komi/
├── index.html
├── artmake.html
├── amarapink.html
├── paramedical.html
├── clinics.html
├── cases.html
├── artists.html
├── contact.html
├── css/
│   └── style.css
├── js/
│   └── main.js
└── images/
    └── （写真ファイル群）
```

> ⚠️ フォルダごとドラッグするとサブフォルダも一緒にアップされる

4. 下部の「Commit changes」をクリック

### 方法B：Git CLIを使う場合（PCにGitがインストール済みの場合）

```bash
# ターミナルでファイルのあるフォルダに移動
cd artmake-komi

# Gitの初期設定
git init
git add .
git commit -m "first commit"

# GitHubと接続してアップ
git branch -M main
git remote add origin https://github.com/（ユーザー名）/artmake-komi.git
git push -u origin main
```

---

## STEP 5｜Cloudflare Pages設定

### 5-1. Cloudflareアカウント作成

1. [pages.cloudflare.com](https://pages.cloudflare.com) にアクセス
2. 「Sign Up」からアカウント作成（無料）

### 5-2. Pagesプロジェクト作成

1. Cloudflareダッシュボードで「Workers & Pages」→「Create」
2. 「Pages」タブ→「Connect to Git」
3. GitHubと連携（「Connect GitHub」）
4. リポジトリ `artmake-komi` を選択
5. ビルド設定：

| 項目 | 設定 |
|------|------|
| Framework preset | **None** |
| Build command | **空欄** |
| Build output directory | **空欄** |

6. 「Save and Deploy」をクリック
7. 数秒〜数分でデプロイ完了
8. `https://artmake-komi.pages.dev` でサイトが見られるようになる

---

## STEP 6｜独自ドメインの接続

### 6-1. Cloudflare側の設定

1. Cloudflare → 対象のPagesプロジェクト → 「Custom domains」
2. 「Set up a custom domain」をクリック
3. `artmake-komi.com` と入力して「Continue」
4. 表示される**ネームサーバー**をメモ（例：`xxx.ns.cloudflare.com`）

### 6-2. お名前ドットコム側の設定

1. お名前.comにログイン
2. 「ドメイン設定」→「ネームサーバーの設定」
3. 「他のネームサーバーを使う」を選択
4. Cloudflareのネームサーバー2つを入力して保存

### 6-3. 反映待ち

- 反映まで最短数分〜最長48時間
- 反映確認：`https://artmake-komi.com` にアクセスしてサイトが表示されればOK
- SSL（https）は**Cloudflareが自動で設定**するので何もしなくていい

---

## STEP 7｜更新方法（公開後）

ファイルを修正したら：

**ブラウザ更新の場合：**
1. GitHubのリポジトリを開く
2. 更新したいファイルをクリック→「鉛筆アイコン（Edit）」
3. 修正して「Commit changes」
4. 自動でCloudflare Pagesに反映（約1〜2分）

**ファイル丸ごと差し替えの場合：**
1. リポジトリで該当ファイルを開く→「...」→「Delete file」でいったん削除
2. 新しいファイルをアップロード

## よくあるトラブル

| 症状 | 原因 | 対処 |
|------|------|------|
| サイトが表示されない | DNS反映待ち | 最大48時間待つ |
| 画像が表示されない | ファイル名・パスのミス | ファイル名が命名規則通りか確認 |
| cssが効いていない | ファイル構成ミス | `css/style.css` のパスを確認 |
| マップが表示されない | iframeブロック | ブラウザの設定を確認 |
